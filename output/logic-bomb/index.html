LOGIC BOMB

A malicious code designed to execute under specific conditions.

Description

A logic bomb is a piece of malicious code that is embedded within a legitimate program or script and is triggered by specific conditions or events. Unlike other types of malware, which may execute immediately upon infection, a logic bomb remains dormant until certain criteria are met. These criteria can include dates, the presence or absence of certain files or user actions. Once activated, a logic bomb can cause various harmful effects, such as deleting files, corrupting data, or compromising system integrity. Logic bombs are often used in insider attacks where an individual with access to the system embeds the code as a form of sabotage or revenge. They can also be used in conjunction with other malware types to enhance their impact. The stealthy nature of logic bombs makes them particularly dangerous, as they can lie undetected within systems for extended periods before triggering, leading to significant damage when eventually activated.

Examples

  • A logic bomb that deletes all files on a specific date, such as an employee's last day of work.
  • A logic bomb that triggers when a specific user logs into a system, causing data corruption.

Additional Information

  • Logic bombs can be difficult to detect because they are often hidden within legitimate software.
  • Preventive measures include regular code reviews and monitoring for unusual system behavior.
